Приложение разделено на три основных слоя:
WPFAppZoo – UI приложения, содержит View и ViewModel приложения и их компоненты. 
Zoo.Domain – содержит доменную модель, Domain Layer.
Zoo.EntityFramework –  позволяет приложению взаимодействовать с базой данных (SQLite), Infrastructure Layer

WpfAppZoo содержит папки:

Commands – содержит команды для взаимодействия с интерфейсом, нажатия кнопок, переключения и обновления вида и т.п.
Components – содержит отдельные компоненты View, их составные части. 
Stores – содержит компоненты, которые загружают, кэшируют и обновляют текущее состояние UI. Например, чтобы не загружать каждый раз базу данных и обеспечить более стабильную работу.
ViewModels – состоит из ViewModel'ей которые связывают графический интерфейс с другими компонентами приложения.
Views – графический интерфейс приложения

ZooDomain cодержит папки:

Commands - содержит CRUD команды(интерфейсы) для взаимодействия доменного слоя приложения (создать, удалить, обновить) с Infrastructure Layer (Entity Framework)
Models – доменные модели (Entities)
Queries  - запросы(интерфейсы) для загрузки объектов доменной модели, взаимодействия  доменного слоя приложения с Infrastructure Layer (Entity Framework)

Zoo.EntityFramework содержит папки:

Commands – CRUD команды, наследуются от интерфейсов доменного слоя (загрузить, удалить, обновить) и обеспечивают взаимодействие c БД через DTO.
DTOs – Data Transfer Objects, передача данных.
Migrations – описывает структуру БД. 
Queries – запросы для взаимодействия с БД, наследуются от интерфейсов доменного слоя.
